# 개인 프로젝트 중간 보고서 (1~3주차)

## 제출물 요약
- 전체 진행률: 60% (최종 완성 기준)
- 1~3주차 추가 기능 요약: 초기 환경 세팅부터 공통 레이아웃 퍼블리싱을 거쳐 현재 카테고리별 동적 라우팅 및 데이터 필터링 기능까지 구현함

## 1. 추가된 기능을 구체적으로 설명했는가
1주차부터 3주차까지 진행하면서 추가 및 발전된 주요 기능들은 다음과 같음

- 1주차 (초기 설정 및 기획)
  - Vite와 React, TypeScript를 활용해 프로젝트 보일러플레이트 세팅함
  - 프로젝트 기획 및 전역 색상 변수, 다크모드 대응 등 공통 CSS 모듈을 초기 설정함
  - 깃허브 레포지토리 연동 및 기본 아키텍처 구상함

- 2주차 (공통 레이아웃 및 정적 뷰 퍼블리싱)
  - 사이트 전반에 쓰이는 공통 컴포넌트(Header, Footer)를 마크업 및 스타일링함
  - 랜딩 페이지(LandingPage)를 만들고 Hero 섹션과 ToolCard 컴포넌트를 조립해 더미 데이터 기반의 리스트 뷰를 완성함
  - 반응형 웹 디자인 기초 공사를 진행함

- 3주차 (라우팅 및 동적 데이터 렌더링)
  - react-router-dom을 도입해 공통 Layout 안에 서브 페이지만 교체되는 SPA 전역 라우팅 환경을 구축함
  - 글쓰기, 디자인, 코딩, 영상 등 카테고리별 페이지를 나누어 URL로 접근할 수 있게 만듦
  - CategoryPage라는 공통 템플릿 컴포넌트를 설계하여 하드코딩 없이 TOOLS 데이터에서 조건에 맞는 항목만 filter()로 걸러내서 동적 렌더링하게 수정함
  - HTML details와 summary 태그를 사용해 카테고리별 맞춤형 FAQ 아코디언 기능을 추가함

- 앞으로의 계획 (4~5주차)
  - 모바일 해상도까지 완벽하게 지원하는 반응형 최적화 진행 예정임
  - 도구별 상세 페이지 뷰 구성 및 데이터 연결 작업 마무리 예정임
  - 최종 테스트 후 Vercel 등을 통한 배포 진행 예정임

## 2. 코드에 주석으로 기능 설명이 충분한가
원활한 코드 리딩과 기능 명세를 위해 핵심 로직 및 컴포넌트에 주석을 꼼꼼하게 추가함
- App.tsx: 전역 SPA 라우팅을 담당하는 최상위 컴포넌트임을 명시하고 404 리다이렉트 예외처리 로직을 주석으로 설명함
- CategoryPage.tsx: FAQ 데이터 선언부 카테고리별 공통 레이아웃 재사용 목적 프롭스(props) 설명 등을 논리적 구간마다 주석으로 분리함
- LandingPage.tsx: 추천 AI 도구와 전체 AI 도구 배열을 어떻게 매핑하여 그리는지 주석으로 명시함

## 3. 이전 보고서 대비 발전된 내용이 있는가
- 초반(1~2주차)에는 페이지 이동이 불가능한 단일 화면 위주의 정적인 웹페이지 마크업 형태였음
- 현재(3주차)는 데이터를 분리하여 관리하고 라우터 URL에 따라 동적으로 상태와 데이터를 화면에 매핑해주는 실제 어플리케이션(SPA)의 구조를 온전히 갖추게 됨
